# soyeon_project

## 1. 프로젝트 소개
쇼핑몰 웹사이트의 전체 구조를 설계하는 것을 목표로 한 React 프로젝트

## 2. 개발 목적
세부 기능 구현보다 전체적인 사이트 구조를 먼저 설계하고,
페이지 간 이동과 카테고리 구조를 React Router를 통해 구현하는 것을 목표

## 3. 구현 내용
- 메인 페이지 구성
- 상단 메뉴 구성
  - Search
  - Signup
  - Cart
  - Wishlist
- 메인 배너 구성
  - Best
  - Shop All
  - Album
  - Brand Story

- Shop All 드롭다운 메뉴 구현
  - Clothing
  - Bags
  - Shoes
  - Home
  - Accessories

- 2단 드롭다운 구조 구현
  - Clothing → Outers, Tees, Tops, Boleros, Knits, Dresses, Pants, Skirts, Sets, Swimmewear
  - Home → Bras, Underwear, Tops, Bottoms
  - Accessories → Jewelry, Socks, Leather Acc, iPhone Cases

- Best 드롭다운 메뉴 구현
  - Best Sellers
  - Last Chance

## 4. 사용 기술
- React
- Vite
- React Router DOM
- CSS

## 5. 프로젝트 구조
- Home 페이지
- Brand Story 페이지
- Album 페이지
- Best 페이지
- Search 페이지

## 6. 느낀 점
처음에는 각 카테고리마다 페이지를 따로 만들었지만,
구조를 단순화하면서 공통 컴포넌트를 활용하는 것이 더 효율적이라는 점을 배웠습니다.
이번 프로젝트를 통해 쇼핑몰 웹사이트의 정보 구조 설계의 중요성을 이해할 수 있었습니다.

## 오늘 작업 내용 정리

교수님의 피드백에 따라 기능 구현보다  
전체적인 디자인 구성을 우선적으로 진행하였다.

가장 먼저 쇼핑몰의 핵심이라고 생각한  
메인페이지 상단 배너(헤더)를 구현하였다.

쓰리타임즈 웹사이트를 참고하여  
전체적인 레이아웃과 구조를 잡았으며,  
초기 구조는 AI의 도움을 받아 구성하였다.  

하지만 그대로 사용하기보다는  
직접 수정하고 보완하는 과정을 반복하였다.

---

## 디자인 구성 및 수정 과정

배너를 구성하는 과정에서  
특히 폰트, 정렬, 간격과 같은 디테일 요소에 집중하였다.

쓰리타임즈에서 사용하는 폰트를 적용해보았고,  
유료이거나 그대로 사용할 수 없는 경우에는  
AI의 도움을 받아 비슷한 폰트를 찾아 적용하였다.

- 일부는 실제 폰트 사용  
- 일부는 유사 폰트 적용  

이렇게 혼합하여 전체적인 분위기를 유지하려고 노력하였다.

---

## UI 디테일 수정 작업

초기 구조를 만든 이후에는  
다음과 같은 세부적인 문제들을 직접 수정하였다.

- 로고 / 메뉴 / 아이콘 정렬 문제
- 가운데 메뉴가 정확히 중앙에 위치하지 않는 문제
- 메뉴 간 간격이 일정하지 않은 문제
- hover 시 색상 및 스타일 적용
- 드롭다운 메뉴 간격 및 위치 문제
- submenu가 너무 멀거나 붙지 않는 문제
- 화살표(>)가 텍스트와 맞지 않는 문제
- 드롭다운 배경이 어색하여 반투명으로 변경

이 과정에서 여러 번 수정과 테스트를 반복하면서  
디자인이 자연스럽게 보이도록 조정하였다.

## 로그인 페이지 구현

메인 페이지 이후에는  
로그인 페이지의 디자인 부분을 구현하였다.

기본적인 UI는 AI의 도움을 받아 구성하였지만,  
메인 페이지와 자연스럽게 이어지도록 직접 수정하였다.

구현 요소:
- email 입력창
- password 입력창
- forgot password 문구
- sign in 버튼
- create account 링크

## 헤더 구조 개선 (공통 컴포넌트)

처음에는 배너를 `main.jsx`에 작성하여  
메인 페이지에서만 보이도록 구현하였다.

하지만 로그인 페이지로 이동했을 때  
배너가 사라지는 문제가 발생하였다.

이 문제를 해결하기 위해 AI를 통해 

- 헤더를 `Header.jsx`로 분리
- 모든 페이지에서 공통으로 사용하도록 구조 변경

모든 페이지에서 동일한 UI가 유지되도록 개선하였다.

### 사용한 프롬프트 및 수정 과정

#### 1. 드롭다운 간격 문제

프롬프트  

css에서 드롭다운 submenu가 너무 멀어지는데 어떻게 수정해야해?


AI 제안  
- left 값과 padding 조정

직접 수정  
- submenu 위치를 1단 메뉴 width 기준으로 이동  
- padding 값을 직접 조정하여 간격 최적화  

---

#### 2. 화살표 정렬 문제

프롬프트  

텍스트랑 화살표(>)가 수직으로 안 맞는데 맞추는 방법 알려줘


AI 제안  
- flex 또는 grid 사용

직접 수정  
- flex 대신 grid 구조 적용  

```css
.dropdown-item > a {
  display: grid;
  grid-template-columns: 1fr 20px;
}
화살표 위치 미세 조정
.dropdown-item > a::after {
  transform: translateY(-1px);
}

6. 헤더 구조

프롬프트

쓰리타임즈 스타일로 만들어줘

AI 제안

기본 header 구조 (로고 / 메뉴 / 아이콘)

직접 수정

가운데 메뉴 중앙 정렬 적용
전체 정렬 및 간격 직접 수정
추가적으로 사용한 질문
메인 페이지에서는 헤더가 보이는데 로그인 페이지에서는 안 보이는 문제
드롭다운 메뉴 클릭이 되지 않는 문제
AI 결과 활용 방식

AI가 생성한 코드를 그대로 사용하는 것이 아니라
다음과 같은 방식으로 수정 및 활용하였다.

구조 이해 후 불필요한 코드 제거
CSS 값 직접 수정 (간격, 정렬, 위치)
원하는 디자인에 맞게 재구성
실제 레이아웃에 맞게 코드 변경

특히 드롭다운 메뉴와 정렬 문제는
여러 번 수정하면서 직접 맞추는 작업을 진행하였다.

느낀 점

이번 작업을 통해 단순히 UI를 구현하는 것보다
디테일을 맞추는 과정이 훨씬 중요하고 어렵다는 것을 느꼈다.

AI를 활용하면 빠르게 구조를 만들 수 있지만,
원하는 결과를 만들기 위해서는
직접 수정하고 비교하는 과정이 필요하다는 것을 알게 되었다.

앞으로는 기능 구현과 데이터 연결까지 확장할 계획이다.
